### BFTS research suggestion adoption (`docs/superpowers/bfts-research-suggestion.md`)

The research suggestion proposed a simpler concurrency model (single `bfts_node` table with `status`/`score`/`claimed_by`, SKIP LOCKED CTE claim, `bfts_expand_one` per-node child workflow, janitor, `ctx.run_agent` as expansion unit). Verdict from the analysis subagent:

| Proposal | Status |
|----------|--------|
| Single `bfts_node` table | **Defer** — breaks the normalized Phase 2/3 schema and Sakana-shape DAO without algorithm gain. |
| SKIP LOCKED CTE claim | **Defer** — would rewrite `_bfts_select.select_next` and lose `debug_prob` / one-node-per-root diversification. Only needed at much higher worker scale. |
| `bfts_expand_one` child workflow | **Adopt now (Design B-lite)** — controller-assigned `node_id`s, fan-out via `ctx.start_workflow`, **keep** existing `select_next` + `expand_node`. Becomes Phase 4h. |
| Multi-worker fan-out within a tree | **Adopt now** — same as above; the controller already computes `num_workers` selections per iteration but expands sequentially. |
| Janitor for stale `in_flight` | **Defer** — only needed with SKIP LOCKED + lease columns. Centaur's `WORKFLOW_WORKER_LEASE_S` recovers workflows; failed children leave partial rows that retry on rerun. |
| Partial index on expandable predicate | **Defer** — additive when/if SKIP LOCKED arrives. |
| `ctx.run_agent` as expansion unit | **Reject** — destroys mid-expansion `ctx.step` resume fidelity and replaces the structured Sakana 5–7-call pipeline. |

**Bonus fidelity finding:** `bfts_tree._to_noderef` hardcodes `is_leaf=True` (`overlay/workflows/bfts_tree.py:101`), weakening Sakana's buggy-leaf filter in `_bfts_select._buggy_leaf_nodes`. Goes into Phase 4g.

## Spec corrections (post-research)

The Phase 0–3 spec corrections (`docs/superpowers/plans/2026-05-25-bfts-on-centaur.md:23-40`) still apply. New corrections from research:

1. **"Move tree controller to `ctx.run_agent`-per-node" rejected.** The structured `_bfts_expand.expand_node` pipeline (per-LLM-call `ctx.step`) gives mid-expansion resume that `ctx.run_agent` does not. Phase 4h adopts `bfts_expand_one` as a *wrapper* around the existing pipeline, not a replacement.
2. **Per-node workspace isolation is a hard prereq for intra-tree parallelism.** Today every expansion in a tree shares `/workspace/working` (`overlay/tools/bfts_executor/client.py:27-33`). Parallel `exec_python` calls race on `runfile.py`, `experiment_data.npy`, and any plot outputs. Phase 4h.1 adds per-node working dirs *before* the child-workflow refactor.
3. **`bfts_hyperparams` is an append-only history, not a mutable singleton.** Each nightly reflection inserts a new row; `bfts_root` reads `ORDER BY effective_from DESC LIMIT 1`. Mirrors the plan sketch (`docs/superpowers/plans/2026-05-25-bfts-on-centaur.md:5549-5558`).
4. **No new tool packages.** Phase 4d's ideation/citation workflows consume the existing `overlay/tools/semantic_scholar` client. Adds one line to its `BIBTEX_PAPER_FIELDS` constant.
5. **Phase 4e is opt-in.** Enabling `agentSandbox.controller.extensions: true` adds CRD controllers but does **not** affect non-BFTS sandboxes. The BFTS executor must NOT lose the additive `bfts-sandbox-egress` NetworkPolicy if Phase 4e ships; bare-Sandbox + manual NP remains the executor path even with a Template defined.

## Task 4g.2: Configurable metric reducer

**Why:** `_bfts_metric.score()` hardcodes `mean()` collapse with first-metric `lower_is_better` direction (`docs/superpowers/research/02-ai-scientist-v2.md` §Gotcha #7). For multi-metric experiments (e.g. accuracy + latency) operators want `min`, `weighted_mean`, or `lexicographic`.

**Files:**
- Modify: `overlay/workflows/_bfts_metric.py`
- Modify: `overlay/workflows/_bfts_config.py`
- Modify: `overlay/workflows/bfts_root.py` + `bfts_tree.py`
- Modify: `overlay/workflows/tests/test_bfts_metric.py`

- [ ] **Step 1: Add reducers**

Extend `_bfts_metric.py` with `score(metric, *, reducer: str = "mean") -> float`. Reducers: `mean` (current behavior, unchanged), `min` (best dataset wins), `weighted_mean` (per-metric weights from `metric["weights"]` if present, else uniform), `lexicographic` (sort metrics, compare in order — returns a comparable tuple-encoded float for the existing `min()` argmax site).

Reducer string lives in `_bfts_config.LLMSettings` (rename to `BFTSSettings` or add a sibling `SearchSettings`).

- [ ] **Step 2: Plumb through Input → resolve → score**

`_bfts_select` and `_bfts_export` call `score(metric, reducer=cfg.metric_reducer)`.

- [ ] **Step 3: Test each reducer**

Add fixtures for two-metric nodes; assert each reducer picks the expected best node.

```python
"""Workflow: expand one BFTS node. Child of bfts_tree.

Wraps the existing _bfts_expand.expand_node pipeline in a separate
workflow run so a parent controller can fan out N expansions per
iteration. Each child gets its own checkpoint stream — mid-expansion
crashes resume cleanly without re-running siblings."""
from __future__ import annotations

import sys
from dataclasses import dataclass, field
from pathlib import Path
from typing import TYPE_CHECKING, Any

sys.path.insert(0, str(Path(__file__).resolve().parent))

if TYPE_CHECKING:
    from api.workflow_engine import WorkflowContext

from _bfts_config import resolve_llm_api_key, resolve_llm_settings
from _bfts_expand import ExpandContext, expand_node
from _bfts_state import mark_buggy_plots, update_node_metric

WORKFLOW_NAME = "bfts_expand_one"


@dataclass
class Input:
    run_id: str
    node_id: str
    sandbox_id: str
    working_dir: str
    parent_node: dict[str, Any] | None
    idea: dict[str, Any] = field(default_factory=dict)
    # Optional model overrides; child inherits from parent's bfts_tree Input.
    llm_api_key_secret: str | None = None
    draft_model: str | None = None
    feedback_model: str | None = None
    vlm_model: str | None = None


async def handler(inp: Input, ctx: "WorkflowContext") -> dict[str, Any]:
    llm = resolve_llm_settings(
        draft_model=inp.draft_model,
        feedback_model=inp.feedback_model,
        vlm_model=inp.vlm_model,
        llm_api_key_secret=inp.llm_api_key_secret,
    )
    llm_api_key = resolve_llm_api_key(llm.llm_api_key_secret)
    pool = ctx._pool

    expand_ctx = ExpandContext(
        sandbox_id=inp.sandbox_id,
        parent_node=inp.parent_node,
        idea=inp.idea,
        llm_api_key=llm_api_key,
        node_id=inp.node_id,
        working_dir=inp.working_dir,
        draft_model=llm.draft_model,
        feedback_model=llm.feedback_model,
        vlm_model=llm.vlm_model,
    )

    result = await expand_node(ctx=ctx, expand_ctx=expand_ctx)

    await ctx.step(
        "update_node",
        lambda: update_node_metric(
            pool,
            node_id=inp.node_id,
            term_out=result["term_out"],
            exec_time_seconds=result["exec_time_seconds"],
            exc_type=result["exc_type"],
            exc_info=result["exc_info"],
            exc_stack=result["exc_stack"],
            metric=result["metric"],
            is_buggy=result["is_buggy"],
            analysis=result["analysis"],
            plan=result["plan"],
            code=result["code"],
            parse_metrics_code=result.get("parse_metrics_code"),
            parse_term_out=result.get("parse_term_out"),
            plot_code=result.get("plot_code"),
            plot_term_out=result.get("plot_term_out"),
        ),
    )
    if "is_buggy_plots" in result:
        await ctx.step(
            "mark_buggy_plots",
            lambda: mark_buggy_plots(
                pool,
                node_id=inp.node_id,
                is_buggy_plots=bool(result["is_buggy_plots"]),
                plot_analyses=result.get("plot_analyses"),
                vlm_feedback_summary=result.get("vlm_feedback_summary"),
            ),
        )

    return {
        "node_id": inp.node_id,
        "is_buggy": result["is_buggy"],
        "stage_name": result.get("stage_name"),
    }
```

```python
"""Workflow: nightly BFTS hyperparameter reflection.

Reads the last N completed bfts_runs, computes summary stats, and
inserts a new bfts_hyperparams row that bfts_root will pick up as
defaults for subsequent runs. Off by default; flip on via
BFTS_REFLECTION_ENABLED env in api.extraEnv.
"""
from __future__ import annotations

import os
import sys
from dataclasses import dataclass
from pathlib import Path
from typing import TYPE_CHECKING, Any

sys.path.insert(0, str(Path(__file__).resolve().parent))

if TYPE_CHECKING:
    from api.workflow_engine import WorkflowContext

from _bfts_config import (
    DEFAULT_DEBUG_PROB, DEFAULT_MAX_DEBUG_DEPTH,
    DEFAULT_NUM_DRAFTS, DEFAULT_NUM_WORKERS,
)
from _bfts_hyperparams import insert_hyperparams, latest_hyperparams


def _env_flag_enabled(name: str) -> bool:
    return str(os.getenv(name, "")).lower() in ("1", "true", "yes")


WORKFLOW_NAME = "bfts_reflection_nightly"
SCHEDULE = {
    "cron": "0 3 * * *",
    "timezone": "UTC",
    "no_delivery": True,
    "enabled": _env_flag_enabled("BFTS_REFLECTION_ENABLED"),
    "catchup_policy": "skip",
}


@dataclass
class Input:
    lookback_runs: int = 50


async def handler(inp: Input, ctx: "WorkflowContext") -> dict[str, Any]:
    pool = ctx._pool

    recent = await ctx.step(
        "load_recent_runs",
        lambda: pool.fetch(
            """
            SELECT run_id, idea_json, config_json, best_node_id, status,
                   created_at, updated_at
            FROM bfts_runs
            WHERE status = 'completed'
            ORDER BY updated_at DESC
            LIMIT $1
            """,
            inp.lookback_runs,
        ),
    )

    if not recent:
        ctx.log("bfts_reflection_skipped", reason="no_completed_runs")
        return {"inserted": False}

    # Simple heuristic v1: keep last-known params; tweak debug_prob up by 0.05
    # if fewer than half of recent runs reached a good node; cap to [0.1, 0.8].
    # Replace with a smarter rule when there's real data.
    prev = await ctx.step("load_latest", lambda: latest_hyperparams(pool))
    debug_prob = (prev["debug_prob"] if prev else 0.5)
    good_count = sum(1 for r in recent if r["best_node_id"])
    if good_count < len(recent) / 2:
        debug_prob = min(0.8, debug_prob + 0.05)
    elif good_count == len(recent):
        debug_prob = max(0.1, debug_prob - 0.02)

    await ctx.step(
        "insert_row",
        lambda: insert_hyperparams(
            pool,
            debug_prob=debug_prob,
            max_debug_depth=DEFAULT_MAX_DEBUG_DEPTH,
            num_drafts=DEFAULT_NUM_DRAFTS,
            num_workers=DEFAULT_NUM_WORKERS,
            metric_reducer=(prev["metric_reducer"] if prev else "mean"),
            notes=f"reflection of {len(recent)} runs; good={good_count}",
        ),
    )

    ctx.log(
        "bfts_reflection_inserted",
        debug_prob=debug_prob,
        recent_runs=len(recent),
        good_count=good_count,
    )
    return {"inserted": True, "debug_prob": debug_prob}
```
