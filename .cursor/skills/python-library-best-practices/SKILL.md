# Python library best practices

These rules govern project layout, packaging, public API design, and the build/quality pipeline. They are invariants, not preferences. Follow them literally.

## Scope — read this first

These rules target **publishable Python libraries** (PyPI-distributed, SemVer'd,
trusted-publish via OIDC). For **internal services** that deploy as containers
(no wheel, no PyPI) the publish/CI/docs sections do not apply and several
design rules are looser. In the centaur-lab repo:

| Topic | Publishable library (below) | Centaur-internal code |
| --- | --- | --- |
| `src/` layout + `py.typed` | Yes | Yes if there's a `pyproject.toml`; flat `api/` package is fine for in-container code |
| `hatch-vcs` + git-tag versioning | Yes | No — hand-bump `version = "0.1.0"` (see `.centaur/services/api/pyproject.toml`) |
| `nox` cross-version + GitHub Actions matrix | Yes | No — pinned `requires-python = ">=3.11"` and CI runs once |
| MkDocs Material + mkdocstrings | Yes | No — Markdown docs live in `docs/` or `.centaur/docs/` |
| PyPI Trusted Publishing | Yes | No — never published |
| Single `MylibError` base + structured-field exceptions at module boundaries | Yes | Plain `raise RuntimeError(...)` / `raise ValueError(...)` is the norm; custom error classes only at the public API surface |
| Discriminated unions over wide dataclasses | Yes | Yes |
| Module-as-namespace, verb-only function names | Yes | Pragmatic — repeating the noun is allowed when it aids the call site (`bootstrap_service_api_keys`, `seed_channel_bootstrap_job`) |
| Sync by default, optional `.aio` submodule | Yes | Mostly async (`asyncpg`, `httpx.AsyncClient`); sync helpers exist where required (e.g. `psycopg` in `db/`) |

**Rule for the agent**: skip the "Build backend / CI / Publish / Docs" sections
when working in a centaur-internal package. Apply the layout, naming, and
typing sections unless the surrounding code already does something else — then
match the project. See `python-style-guide.md` §`Scope` for the full table.

## Project layout

**Invariant.** Use the `src/` layout. Tests live in `tests/` at the repo root, without an `__init__.py`. `pyproject.toml` is the only configuration file.

```
mylib/
├── .github/workflows/{ci.yml,publish.yml}
├── .python-version
├── .gitignore
├── pyproject.toml
├── uv.lock              # committed
├── noxfile.py
├── README.md
├── LICENSE
├── CHANGELOG.md
├── src/mylib/
│   ├── __init__.py      # public API surface
│   ├── py.typed         # PEP 561 marker (required)
│   ├── _version.py      # generated by hatch-vcs (gitignored)
│   ├── _core.py         # private modules underscored
│   ├── exceptions.py
│   ├── nn/__init__.py
│   ├── train/__init__.py
│   ├── data/
│   │   ├── __init__.py  # sync API
│   │   └── aio.py       # OPTIONAL async wrapper
│   └── checkpoint/
│       ├── __init__.py  # sync API
│       └── aio.py       # OPTIONAL async wrapper
├── tests/
│   ├── conftest.py
│   └── test_*.py
└── docs/mkdocs.yml
```

The src layout forces installation before import, so tests run against the installed package. Flat layouts mask broken packaging.

## Package manager: `uv`

**Invariant.** Every dependency and environment operation goes through `uv`. Never call `pip` directly. Never use `poetry`, `pdm`, or `pip-tools`.

```bash
uv init --lib --build-backend hatchling mylib

uv add "jax>=0.4.30" "jaxtyping>=0.2.34" "equinox>=0.11" "optax>=0.2" "orbax-checkpoint>=0.6"
uv add --group test  pytest pytest-cov hypothesis chex anyio
uv add --group lint  ruff
uv add --group typecheck pyright
uv add --group docs  mkdocs-material "mkdocstrings[python]"
uv add --optional cuda "jax[cuda12]"

uv lock
uv sync --all-groups --all-extras
uv run pytest
uv build
uv publish    # only in CI; uses trusted publishing
```

## Build backend: `hatchling` + `hatch-vcs`

**Invariant.** `hatchling` is the build backend. `hatch-vcs` derives the version from git tags. Do not commit a hand-edited version string.

```toml
[build-system]
requires      = ["hatchling>=1.27", "hatch-vcs>=0.5"]
build-backend = "hatchling.build"
```

`uv_build` is rejected because it does not support build hooks, and we require VCS-based versioning.

## Versioning

**Invariant.** Versions come from annotated git tags via `hatch-vcs`. Follow SemVer. Releases are `git tag vX.Y.Z && git push --tags`; CI publishes.

In `src/mylib/__init__.py`:

```python
from mylib._version import __version__   # generated by hatch-vcs
```

In CI, `actions/checkout` must set `fetch-depth: 0` and `fetch-tags: true`.

## `__init__.py` and the public API

**Invariant.** The public API is exactly what `mylib/__init__.py` (and the `__init__.py` of public subpackages) re-exports inside `__all__`. Anything else has no SemVer guarantee.

Rules:

1. **Private modules begin with `_`** (`_core.py`, `_internal/`). Users importing these are off-warranty.
2. `__init__.py` is **minimal**: re-exports only. No computation, no side effects.
3. Use **`from .x import Name as Name`** so pyright strict accepts the re-export.
4. **`__all__`** is alphabetically sorted and lists every public symbol the module exposes.
5. **No wildcard imports**, anywhere.

```python
# CORRECT — src/mylib/__init__.py
"""mylib — JAX-based ML research library."""
from mylib._version import __version__
from mylib._core import Model as Model
from mylib._core import train_step as train_step
from mylib.exceptions import ConfigError as ConfigError
from mylib.exceptions import MylibError as MylibError
from mylib import checkpoint as checkpoint
from mylib import data as data
from mylib import nn as nn
from mylib import train as train

__all__ = [
    "ConfigError",
    "Model",
    "MylibError",
    "__version__",
    "checkpoint",
    "data",
    "nn",
    "train",
    "train_step",
]
```

```python
# INCORRECT
from mylib._core import *
from mylib._core import Model         # implicit re-export
DEFAULT = expensive_compute()           # side effect at import
```

For deprecation, use a module-level `__getattr__`:

```python
def __getattr__(name: str) -> object:
    if name == "OldName":
        import warnings
        warnings.warn("OldName is deprecated; use NewName.", DeprecationWarning, stacklevel=2)
        from mylib._core import NewName
        return NewName
    raise AttributeError(name)
```

## Module organisation

**Invariant.** Library users see three tiers:

1. **Top-level symbols** (`mylib.Model`, `mylib.train_step`) — the most common entry points.
2. **Public subpackages** (`mylib.nn`, `mylib.train`, `mylib.checkpoint`) — themed groupings.
3. **Private modules** (`mylib._core`, `mylib._internal.*`) — implementation; never imported by users.

Python has no `import * as Foo`. The equivalent is `from mylib import nn` (imported subpackage acts as a namespace handle). Provide that by adding the subpackage to top-level `__all__`.

### What every domain module exports

**Invariant.** A *domain* module (the meat of the library: `mylib.optim`, `mylib.train`, `mylib.checkpoint`, `mylib.data` — anything users call directly) exports exactly four kinds of symbols, in this order:

1. **Types** — the core dataclass(es), `Literal` aliases for discriminated unions, `Protocol`s for interfaces consumed across modules.
2. **Functions** — pure transformers with verb-only names (`create`, `update`, `step`, `sample`, `entropy`). The module already carries the noun; do not repeat it. See `python-style-guide.md` §`API design and naming` for the naming rules.
3. **Errors** — module-specific exception subclasses inheriting from the library base.
4. **Constants** — module-level `Final[T] = ...` for fixed numeric defaults. Nothing else runs at import time.

```python
# src/mylib/optim.py
"""SGD-with-momentum optimiser."""

from dataclasses import dataclass, replace
from typing import Final

import jax
import jax.numpy as jnp

from mylib.exceptions import MylibError

# 1. Types
@dataclass(frozen=True, slots=True, kw_only=True)
class State:
    params: jax.Array            # current parameter values
    velocity: jax.Array          # exponential moving average of grads
    step: int                    # number of update steps so far

@dataclass(frozen=True, slots=True, kw_only=True)
class Config:
    learning_rate: float
    momentum: float = 0.9
    weight_decay: float = 0.0

# 2. Functions — verb-only; operate on State
def create(config: Config) -> State: ...
def step(state: State, *, grads: jax.Array) -> State: ...
def schedule(state: State) -> jax.Array: ...
def zero_grad(state: State) -> State: ...

# 3. Errors
class NotFiniteError(MylibError, ValueError):
    """Raised when gradients or parameters contain NaN or Inf values."""

# 4. Constants
MAX_GRAD_NORM: Final[float] = 1.0
```

Re-export the module as a namespace in `src/mylib/__init__.py`:

```python
from mylib import optim as optim

__all__ = [..., "optim"]
```

Users then write `mylib.optim.step(state, grads=g)` or `from mylib import optim; optim.step(state, grads=g)`. The single-file module is the unit; only split into a subpackage (`mylib/optim/__init__.py` plus `_internal/`) when the file grows past ~400 lines or needs its own private helpers.

Utility, infrastructure, and shared-type modules (`mylib.exceptions`, `mylib._internal.linalg`) are exempt from this exact shape — they exist to *support* domain modules, not to be domain modules themselves.

## Functions, classes, factories

**Invariant.** Prefer plain functions for behaviour and frozen dataclasses for data. Use classes only for genuine state across calls or for typed hierarchies (exceptions, `Protocol`, `eqx.Module`).

Composition over inheritance. Subclassing is justified only for: (a) abstract interface implementation, (b) exception hierarchies, (c) `eqx.Module` (which is itself a frozen dataclass).

```python
# CORRECT — data is frozen, behaviour is a verb-only function in a noun module
# src/mylib/train.py
@dataclass(frozen=True, slots=True, kw_only=True)
class Config:
    learning_rate: float
    batch_size: int
    steps: int

def create(config: Config) -> optax.GradientTransformation: ...
def step(state: State, *, batch: Batch) -> tuple[State, Metrics]: ...

# Callsite
from mylib import train
opt = train.create(train.Config(learning_rate=1e-3, batch_size=32, steps=10_000))
state, metrics = train.step(state, batch=batch)

# INCORRECT — wrapping a pure transform in a stateful "manager"
class Trainer:
    def __init__(self, config: Config) -> None:
        self.config = config
    def step(self, state, batch):
        ...
```

Constructors with non-trivial logic become module-level **factory functions** — never `@classmethod`s. Use the canonical names:

| Source                        | Function                                              |
| ----------------------------- | ----------------------------------------------------- |
| A typed `Config` object       | `create`                                              |
| A serialized blob             | `from_bytes`, `from_dict`, `from_json`, `from_path`   |
| Loose positional fields       | `make`                                                |

`from` alone is a Python keyword, so always use the `from_<source>` suffix. Avoid ad-hoc names (`build`, `construct`, `init`, `new`) — the table above covers every case. See `python-style-guide.md` §`API design and naming` for the rationale and the full naming-rule table.

## Sync vs async

**Invariant.** The library is **sync by default**. Every JAX-touching path is sync. See `jax-best-practices.md` — JAX itself is sync-native.

Async is permitted **only** for pure I/O at the periphery: checkpoint upload/download, dataset streaming from remote stores, telemetry. When async is offered, expose it as a separate submodule (`mylib.checkpoint.aio`, `mylib.data.aio`). The implementation uses `anyio` (works on asyncio and trio), never bare `asyncio`.

Pattern: a sans-I/O core + thin sync wrapper + optional async wrapper that offloads blocking work with `anyio.to_thread.run_sync`.

```python
# mylib/checkpoint/_io.py — sans-I/O core
def serialize_state(state: TrainState) -> bytes: ...

# mylib/checkpoint/__init__.py — sync API (default)
def save(state: TrainState, path: Path) -> None:
    path.write_bytes(serialize_state(state))

# mylib/checkpoint/aio.py — OPTIONAL async wrapper
import anyio
async def save(state: TrainState, path: anyio.Path) -> None:
    blob = serialize_state(state)
    await path.write_bytes(blob)
```

When to add an async API:

- The operation is I/O-bound (network, disk on a remote FS).
- A measured workload benefits from concurrency.
- The sync version already exists, works, and is the source of truth.

Do **not** create async variants speculatively. Do **not** mark JAX-touching functions `async`. Use Orbax's `AsyncCheckpointer` for non-blocking checkpoint writes — do not wrap JAX calls in your own coroutines.

## Parameter conventions

**Invariant.**

- **Keyword-only `*`** for nearly all parameters beyond one or two obvious positionals.
- **Positional-only `/`** when the parameter name is an implementation detail or likely to clash with `**kwargs`.
- **No mutable default values.** Use `None` + assign inside, or `field(default_factory=...)`.
- **Sentinels** when `None` is a meaningful value: `_UNSET: Final = object()`.
- **Use a frozen dataclass** for >4 related parameters; do not let signatures grow indefinitely.
- **Type `**kwargs`** with `TypedDict` + `typing.Unpack` (3.12+).

```python
# CORRECT
from typing import TypedDict, Unpack

class TrainKwargs(TypedDict, total=False):
    lr: float
    weight_decay: float

def train(
    model: Model,
    /,
    *,
    dataset: Dataset,
    seed: int = 0,
    **kwargs: Unpack[TrainKwargs],
) -> Model: ...

# INCORRECT
def train(model, dataset, lr=0.001, weight_decay=0.0, batch_size=32, steps=1000): ...
def f(items=[]):                    # mutable default
    items.append(1)
    return items
```

## Generics

**Invariant.** PEP 695 inline syntax (Python 3.12+) for new code. `TypeVar(...)` is forbidden in new modules.

```python
# CORRECT
class Container[T]:
    def __init__(self, item: T) -> None:
        self.item = item

def first[T](xs: list[T]) -> T:
    return xs[0]

type Pair[T] = tuple[T, T]

def trace[**P, R](fn: Callable[P, R]) -> Callable[P, R]: ...
```

`Self` for fluent and clone-returning methods. `Protocol` for parameter types in public signatures (accept anything that looks like X). `Generic` (via `class Foo[T]:`) when the implementation is genuinely parameterised by type. Prefer `Protocol` for everything user-facing.

For arrays, use `jaxtyping` annotations (`Float[Array, "batch dim"]`) — see `jax-best-practices.md`.

## Immutability

**Invariant.** All data classes are `frozen=True, slots=True, kw_only=True`. Update with `dataclasses.replace(...)`. Stateful objects (rare) are explicit and documented.

```python
@dataclass(frozen=True, slots=True, kw_only=True)
class ModelConfig:
    hidden_dim: int
    n_layers: int
    dropout: float = 0.0

new_cfg = replace(cfg, dropout=0.1)
```

Pydantic models at trust boundaries are also frozen:

```python
class Settings(BaseModel):
    model_config = ConfigDict(frozen=True, extra="forbid")
    lr: float
    seed: int = 0
```

### No input mutation

**Invariant.** Functions never mutate their inputs. Return a new value built with `dataclasses.replace`, JAX's `.at[idx].set(...)`, or `np.array(arr, copy=True)`. `frozen=True` makes this a runtime error for dataclasses; arrays, dicts, and lists have no such guard, so the rule is the only protection.

```python
# CORRECT — returns new state
def step(state: State, *, grads: jax.Array, learning_rate: float) -> State:
    params = state.params - learning_rate * grads
    return replace(state, params=params, step=state.step + 1)

# INCORRECT — mutates an ndarray argument in place
def normalise(xs: np.ndarray) -> None:
    xs /= xs.sum()                              # caller's array is now modified

# CORRECT — return a new array
def normalise(xs: np.ndarray) -> np.ndarray:
    return xs / xs.sum()

# CORRECT (JAX) — functional update via .at
def zero_first(xs: jax.Array) -> jax.Array:
    return xs.at[0].set(0.0)
```

Reasons: pure functions are JAX-traceable, parallelisation-safe, snapshot-testable, and trivially memoised. Hidden mutation is the most common source of subtle JAX bugs (silent retraces, NaN propagation through `jit` caches, surprising aliasing under `vmap`).

## Error handling

**Invariant.**

1. **One base exception per library**: `class MylibError(Exception): ...`. All custom errors inherit from it.
2. Chain with `raise X from Y`. Use `from None` only to intentionally hide the cause.
3. **Exceptions, not Result types.** Python idiom.
4. **`ExceptionGroup`** (PEP 654) only when actually raising multiple unrelated errors at once (parallel I/O, batched validation). Do not retrofit existing single-exception APIs.
5. Multi-inherit from a stdlib class when semantically useful (`ConfigError(MylibError, ValueError)`).
6. Build error messages in a local variable.

```python
# src/mylib/exceptions.py
class MylibError(Exception):
    """Base class for all mylib errors."""

class ConfigError(MylibError, ValueError):
    """Raised when user configuration is invalid."""

class ShapeError(MylibError, ValueError):
    """Raised when array shapes are incompatible."""

class CheckpointError(MylibError, OSError):
    """Raised when a checkpoint cannot be read or written."""

# Usage
try:
    raw = load(path)
except OSError as exc:
    msg = f"Could not load checkpoint at {path!r}"
    raise CheckpointError(msg) from exc
```

### Structured fields on exceptions

**Invariant.** Exceptions raised across a module boundary carry typed fields for programmatic handling, not just a message string. The message is for humans; the fields are for callers.

```python
class ShapeError(MylibError, ValueError):
    """Raised when an array shape is incompatible with an operator.

    Attributes:
        expected: The shape the operator required.
        actual:   The shape that was supplied.
    """

    def __init__(
        self,
        msg: str,
        *,
        expected: tuple[int, ...],
        actual: tuple[int, ...],
    ) -> None:
        super().__init__(msg)
        self.expected = expected
        self.actual = actual

# Raise
if grads.shape != state.params.shape:
    msg = f"gradient shape {grads.shape} != parameter shape {state.params.shape}"
    raise ShapeError(msg, expected=state.params.shape, actual=grads.shape)

# Catch — typed access to structured fields
try:
    step(state, grads=grads)
except ShapeError as exc:
    log.error("shape mismatch", extra={"expected": exc.expected, "actual": exc.actual})
```

`@dataclass` does NOT compose with `Exception` cleanly (multi-inheritance, `slots`, and `__init__` interact badly). Vanilla `__init__` with keyword-only fields is the simplest pyright-strict-clean option. Document each field in the class docstring's `Attributes:` block so mkdocstrings renders it.

This pattern is for *typed contract* errors raised across module boundaries (shape, config, checkpoint, I/O). Pure-message exceptions (`AssertionError`, transient log-only warnings) do not need it.

#### Alternative: generic structured-context kwargs

When the field set varies per call site (one `ShapeError` carries `expected`/`actual`; another also wants `op="factor_dot"`; a third just wants `axis=2`), the per-class typed `__init__` becomes restrictive — every new context field requires a new optional kwarg or a new subclass. Use a single `**context` kwarg on the base class instead:

```python
import sys

if sys.version_info >= (3, 12):
    from typing import override
else:
    from typing_extensions import override


class MylibError(Exception):
    """Base class for all mylib errors.

    Args:
        message: Human-readable diagnostic. Stored on ``self.args[0]``
            via :class:`Exception` and rendered first in ``str(exc)``.
        **context: Arbitrary structured context (shapes, expected/actual
            values, originating operation name, etc.). Stored on
            ``self.context`` and appended to ``str(exc)``.
    """

    def __init__(self, message: str, /, **context: object) -> None:
        super().__init__(message)
        self.context: dict[str, object] = context

    @override
    def __str__(self) -> str:
        base = super().__str__()
        if not self.context:
            return base
        rendered = ", ".join(f"{k}={v!r}" for k, v in self.context.items())
        return f"{base} [{rendered}]"


class ShapeError(MylibError, ValueError):
    """Raised when array shapes are incompatible."""


# Raise — context is opt-in and varies per site
raise ShapeError("ndim mismatch", expected=3, actual=4, op="factor_dot")
raise ShapeError("axis out of range", axis=2, ndim=2)

# Catch — fields live on the dict, not as named attributes
try:
    factor_dot(m, xs)
except ShapeError as exc:
    log.error("shape mismatch", extra=exc.context)
```

Trade-offs vs. the typed-fields pattern above:

| | Typed fields (canonical) | Generic context (alternative) |
|---|---|---|
| Field access | `exc.expected` | `exc.context["expected"]` |
| Pyright autocomplete on `exc.X` | ✅ yes | ❌ no |
| Mistype protection at call site | ✅ yes | ❌ no (any kwarg accepted) |
| Backwards-compat with bare `raise Error("msg")` | ❌ requires kwargs | ✅ kwargs are optional |
| Per-subclass `__init__` boilerplate | ✅ one per subclass | ❌ zero (inherited) |
| Field set fixed across raise sites | ✅ declared once | ❌ varies freely |

When to pick which:

- **Default to the typed pattern** for cross-module errors with a fixed, stable field set (the canonical case — most shape/config/checkpoint errors).
- **Use the generic pattern** when (a) the field set genuinely varies per call site, (b) you need to preserve `raise Error("msg")` and `warnings.warn("msg", WarningClass)` call shapes for back-compat with already-shipped code, or (c) one-line subclasses are worth more than typed introspection (libraries with deep exception hierarchies and shallow per-class state).

The two patterns are not mixable inside one hierarchy — pick one per library base class. Switching later is a breaking change for any caller doing `exc.expected` vs `exc.context["expected"]`.

## Dependency rules

**Invariant.**

- **Minimal runtime deps.** Every added dependency is a tax on every user.
- **Lower bounds only.** `>=` is required when you actually use a feature added in that version. **Never** upper-cap library runtime deps (`<2`, `~=`, `==` are forbidden in `[project].dependencies`).
- **Optional extras** (`[project.optional-dependencies]`) for consumer-facing variants: `cuda`, `tpu`, plotting.
- **Dev tools** go in `[dependency-groups]` (PEP 735) — they are **not** published.
- Application/notebook code pins exactly via the committed `uv.lock`.

```toml
# CORRECT
dependencies = ["jax>=0.4.30", "numpy>=1.26"]

# INCORRECT
dependencies = ["jax>=0.4.30,<0.5", "numpy~=1.26"]
```

## Quality tooling

**Invariant.**

- **`ruff`** is the single lint+format tool. No black, no isort, no flake8, no pyupgrade.
- **`pyright --strict`** is the type-check gatekeeper.
- **`pytest`** with `--strict-markers`, `--strict-config`, `--import-mode=importlib`, `filterwarnings = ["error"]`, branch coverage.
- **`nox`** is the cross-version task runner. Tox is forbidden.
- **CI is the source of truth.** Pre-commit hooks are optional convenience for developers; CI re-runs everything.

```python
# noxfile.py
import nox

PYTHONS = ["3.11", "3.12", "3.13"]
nox.options.default_venv_backend = "uv"

@nox.session(python=PYTHONS)
def tests(session):
    session.run_install(
        "uv", "sync", "--locked", "--group", "test",
        env={"UV_PROJECT_ENVIRONMENT": session.virtualenv.location},
    )
    session.run("pytest", *session.posargs)

@nox.session
def lint(session):
    session.install("ruff")
    session.run("ruff", "check", ".")
    session.run("ruff", "format", "--check", ".")

@nox.session
def typecheck(session):
    session.run_install(
        "uv", "sync", "--locked", "--group", "typecheck",
        env={"UV_PROJECT_ENVIRONMENT": session.virtualenv.location},
    )
    session.run("pyright", "src", "tests")
```

## CI/CD

**Invariant.** GitHub Actions only. `astral-sh/setup-uv@v8` for environment setup. PyPI publishing uses **Trusted Publishing** (OIDC) — no API tokens stored as secrets.

```yaml
# .github/workflows/ci.yml
name: CI
on:
  push:        { branches: [main] }
  pull_request:
  workflow_dispatch:

permissions: { contents: read }

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  test:
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest]
        python: ["3.11", "3.12", "3.13"]
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0, fetch-tags: true }
      - uses: astral-sh/setup-uv@v8
        with:
          enable-cache: true
          python-version: ${{ matrix.python }}
      - run: uv sync --locked --all-extras --all-groups
      - run: uv run ruff check --output-format=github .
      - run: uv run ruff format --check .
      - run: uv run pyright src tests
      - run: uv run pytest --cov=mylib --cov-branch --cov-report=xml
      - uses: codecov/codecov-action@v4
        with: { files: coverage.xml }
```

```yaml
# .github/workflows/publish.yml
name: Publish
on:
  push: { tags: ["v*"] }

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0, fetch-tags: true }
      - uses: astral-sh/setup-uv@v8
      - run: uv build
      - uses: actions/upload-artifact@v4
        with: { name: dist, path: dist/ }

  publish:
    needs: build
    runs-on: ubuntu-latest
    environment: { name: pypi, url: https://pypi.org/p/mylib }
    permissions: { id-token: write }
    steps:
      - uses: actions/download-artifact@v4
        with: { name: dist, path: dist/ }
      - uses: pypa/gh-action-pypi-publish@release/v1
```

PyPI side: project → Publishing → Add trusted publisher; provider `GitHub`, workflow `publish.yml`, environment `pypi`.

## Quick-reference checklist

1. `src/` layout, `tests/` outside the package, no `tests/__init__.py`.
2. `hatchling` + `hatch-vcs`; version from git tags.
3. `uv` for every dependency operation; `uv.lock` committed.
4. `pyproject.toml` only; PEP 621 metadata; PEP 735 dependency groups; lower bounds only on runtime deps.
5. `py.typed` shipped.
6. Public API = `__all__` of top-level `__init__.py` and public subpackages; private modules underscored.
7. `__init__.py` is re-exports only — no logic, no side effects.
8. Prefer functions for behaviour, frozen dataclasses for data; subclass only when truly justified.
9. **Sync by default**; async only for pure I/O, in a separate submodule, via `anyio`.
10. Keyword-only parameters by default; no mutable defaults; `TypedDict`+`Unpack` for `**kwargs`.
11. PEP 695 generics; `Protocol` for interfaces; `Self` for fluent APIs.
12. `frozen=True, slots=True, kw_only=True` on every dataclass.
13. Single exception base `MylibError`; always `raise X from Y`; messages built in a local variable.
14. `ruff` + `pyright --strict` + `pytest` (branch coverage) + `nox`; CI is source of truth.
15. Docs: `mkdocs-material` + `mkdocstrings[python]`, Google docstrings.
16. CI: GitHub Actions + `astral-sh/setup-uv@v8`; publish via PyPI Trusted Publishing.
17. Every domain module exports exactly four kinds of symbol: types, verb-only functions, module-specific errors, `Final` constants. Re-exported as a namespace from the top-level `__init__.py`.
18. Factory functions follow canonical names: `create(config)`, `from_bytes/dict/json/path(...)`, `make(...)`. No heavy work in `__init__`.
19. Functions never mutate inputs. Use `dataclasses.replace`, `.at[].set(...)` (JAX), or `np.array(..., copy=True)` to return new values.
20. Exceptions crossing a module boundary carry typed fields (vanilla `__init__` with keyword-only attributes); message text is for humans only.
